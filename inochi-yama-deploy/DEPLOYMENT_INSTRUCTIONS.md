# 命の山を守る会 ウェブサイト デプロイ手順

このファイルでは、「命の山を守る会」ウェブサイトをデプロイするための手順を説明します。

## ファイル構成

- `index.html` - メインのウェブサイトファイル
- `scene1.png` - 背景画像（ASCII名）
- `シーン１.png` - 背景画像（日本語名）
- `netlify.toml` - Netlify設定ファイル

## デプロイオプション

### オプション1: Netlify（推奨）

Netlifyは静的ウェブサイトを無料で簡単にホストできるサービスです。

1. [Netlify](https://app.netlify.com/)にアカウント登録（GitHubアカウントでログイン可能）
2. ダッシュボードから「New site from Git」をクリック
3. GitHubを選択し、リポジトリを連携
4. デプロイ設定で特に変更は不要（netlify.tomlが設定済み）
5. 「Deploy site」をクリックしてデプロイ

もしくは、直接ファイルをアップロードする方法:

1. [Netlify](https://app.netlify.com/)にログイン
2. 「Sites」タブをクリック
3. サイト一覧画面に全てのファイルをドラッグ＆ドロップ
4. 自動的にデプロイが開始され、数秒後にサイトが公開される

### オプション2: GitHub Pages

1. GitHub上に新しいリポジトリを作成（例：`username.github.io/inochi-yama`）
2. リポジトリをクローン `git clone https://github.com/username/inochi-yama.git`
3. ファイルをリポジトリフォルダにコピー
4. 変更をコミットしプッシュ:
   ```
   git add .
   git commit -m "Initial website deployment"
   git push origin main
   ```
5. リポジトリの設定から「Pages」を開き、「main」ブランチからサイトを公開する設定にする

### オプション3: レンタルサーバー/ホスティングサービス

お持ちのレンタルサーバーやホスティングサービスがある場合:

1. FTPクライアントを使ってファイルをアップロード
2. index.htmlがドキュメントルートに配置されるようにする

## 注意点

- YouTube動画の埋め込みが正常に表示されない場合は、以下を確認してください:
  - インターネット接続が有効か
  - ブラウザでCookieやサードパーティコンテンツがブロックされていないか
  - Webサーバーが適切なCORS設定をしているか

- 背景画像が表示されない場合:
  - `scene1.png`と`シーン１.png`の両方がサーバーにアップロードされていることを確認
  - Webサーバーが日本語ファイル名を正しく処理できるか確認

## カスタマイズ

サイトをカスタマイズする場合は、以下のファイルを編集してください:

- `index.html` - サイト全体の内容とスタイル
- YouTubeの動画を変更する場合は、iframe要素の`src`属性のURLを更新してください
